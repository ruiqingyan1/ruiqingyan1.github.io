部署Hexo个人博客到阿里云ECS云服务（其他服务器同样适用）

发表于2024-03-01|更新于2024-03-21|兴趣课Blog全程搭建
|字数总计:1.5k|阅读时长:5分钟|阅读量:

🖼 所有搭建博客的汇总在这里：Blog全程搭建

🎑简单说明

我用的是最新的Centos Stream 9镜像，不同于Centos7.9的yum包管理，用的是dnf

（我在使用Centos7.9的时候出现—操作系统版本过低的问题，所以过度到了Centos Stream）

🌁 一般来说，部署到服务器有三种推送模式

1、直接在服务器本地推送（不方便）

2、在电脑本地推送到服务器，服务器本地建一个钩子自己推送到本地（本文）

3、以github为中心，每次推送到github，服务器定时从github同步项目到本地（宝塔建一个计划任务就ok）

🌉其中，我们需要用到一些工具

1、finalshell，初次ssh链接服务器用到的，全平台通用的软件（很方便，包括ftp上传下载文件）

2、宝塔Linux，可以理解为一键安装配置的，便捷操作服务器的一款在线软件（之后操作linux也可以直接在宝塔）

3

🌺第一步：购买、配置、测试服务器

1、购买服务器

1、在中国备案需要服务器需要服务器在中国大陆，国外地区无法在国内备案

2、买其他服务器也可以，自己选个便宜的就OK，2核2G3M完全够用（推荐阿里云）

活动截止到2026.3.31：https://www.aliyun.com/lowcode/promotion/allinaliyun/99program

Centos7.9 对于node的支持只到v16，有些hexo包用不了，所以不用

Centos8.4 可以使用，但是官方已停止服务，所以没用

Centos Stream 9，宝塔Linux不支持，所以不用

2、配置服务器

设置实例密码，这里的实例指的就是我们的云服务器

进入控制台（一般在右上角头像旁边，或者上面直接搜控制台到控制台首页）—>

点击我的资源里的云服务器ECS进入云服务器ECS控制台—>

进入我们刚购买的ECS实例管理—>

找到重置密码选项—>

设置后确认

3、测试服务器

测试服务器是否能远程登录，各操作系统都可以用finalshell（下载可能需要VPN代理）

注：此时自己的服务器应该是是运行中

🌼第二步：安装宝塔面板（CentOS）

1、使用命令安装宝塔服务

这里会随意分配一个端口给宝塔面板（这里是18188），

我们需要去我买的ecs云服务器配置界面安全组，把这个端口放行才能使用，

这里的账号密码一定要复制保存下来！！！不然后续再装很麻烦

2、配置ecs云服务器安全组，使其可以正常访问宝塔面板

如果网站已备案，用域名代替公网IP即可进入网站，比如ruiqingyan.com是我的域名

未备案时—外网面板地址: https://120.79.58.242:18188/c6bf7972

备案后（DNS解析后，域名换成自己的）—外网面板地址: https://ruiqingyan.com:18188/c6bf7972

3、使用宝塔配置Centos Strem 9环境

进入后需要先同意协议 ，之后绑定登录自己的账号 （免费，放心注册）

推荐安装LNMP（L是Linux，N是Nginx，M是MySQL，P是PHP），等待大概半个小时

初次进入才会有这个弹窗，直接编译安装LNMP，需要30min~2h（此时可以去看看网站备案）

只选中Nginx就可以了

这里会重新在centos上安装宝塔（访问地址不变，重置账号密码）

并下载所有软件

🌸第三步：购买域名、域名备案

购买域名看这里（其实不用看，进去一步步操作就行）

备案，需要打印承诺书、签字、并按手印（指纹就ok）

有点麻烦，个人感觉就是是网站更正规一点，但是用处不大

🏵第四步：安装hexo

1、在宝塔linux页面一键安装node（并配置一下环境变量，在宝塔 和 ecs 两处 都需要配置）

2、npm下载cnpm（测试），再用cnpm下载其他模块（npm镜像在国外，cnpm镜像在国内）

1、安装node，并配置环境

（3）中需要用v20及以上版本的node（镜像要是Centos8.4，如果是Centos7.9只支持到v16）

（4）不配置命令行版本号，ecs不能正常使用node

2、安装cnpm

可以在finalshell，也可以在宝塔Linux的终端界面直接进行

[root@iZwz9axdw3792zbyft06upZ ~]# npm config set registry https://registry.npmjs.org/
[root@iZwz9axdw3792zbyft06upZ ~]# npm install -g hexo-cli

如果没有显示版本信息，需要在“/etc/profile”里添加npm的安装路径的bin文件地址，比如我这里就是需要添加“export PATH=$PATH:/www/server/nodejs/v20.10.0/bin”，使用“npm config get prefix”可以查看npm的安装路径。

换源。先使用“cnpm config get registry”获得当前源，看看是不是淘宝源“https://registry.npm.taobao.org”，不是的话改成淘宝的。

🌻第五步：在ecs部署hexo

1、开放hexo端口（在阿里云和宝塔任一放行端口即可？💮💮💮💮💮），本地部署远程访问测试

2、去除端口号远程访问（不需要类似4000的端口号，公网IP直接访问）

3、https协议安全访问网站（正常部署是http，访问会提示不安全，保密性差容易被攻击）

一般ssl各大网站都有，阿里云的ssl挺贵的，但是有三个月免费使用，我用的是ISRG的Let’s Encrypt，长期免费，90天一个周期自动续费？

1、安装hexo包

2、初始化hexo，本地运行测试

3、放开端口，远程测试

4、通过域名直接访问，取消端口号

5、https协议，网页安全

🎄第六步：钩子，mac推送服务器，服务器自动推送本地Nginx
